## 🎲 使用虚拟数据生成器

为了快速测试系统功能，建议使用管理员账户的"生成测试数据"功能：

### 操作步骤

1. **登录管理员账户**
   ```
   用户名: admin
   密码: admin123
   ```

2. **进入数据生成界面**
   - 点击"🎲 生成测试数据"按钮

3. **配置生成参数**
   - 用户数量: 10-50 个（推荐20个）
   - 商品数量: 30-100 件（推荐50件）
   - 订单数量: 10-100 个（推荐30个）

4. **开始生成**
   - 点击"开始生成"按钮
   - 观察日志输出
   - 等待生成完成

5. **查看生成结果**
   - 用户管理：查看新增用户
   - 商品管理：查看发布的商品
   - 订单管理：查看生成的订单
   - 数据看板：查看统计图表

### 生成的数据特点

#### 用户数据
- 用户名：`user_随机4位数字`（例如：user_1234）
- 密码：统一为 `123456`
- 初始余额：随机 ¥100-¥5000
- 联系方式：随机手机号

#### 商品数据
- 名称：从预设商品库随机选择
- 类别：学习资料、电子产品、生活用品、运动器材、服饰鞋包、其他
- 价格：随机 ¥10-¥500
- 描述：自动生成（包含新旧程度）
- 卖家：从生成的用户中随机选择

#### 订单数据
- 自动匹配买卖双方
- 确保买家有足够余额
- 自动完成交易流程
- 状态：已完成

## 🔒 安全说明

### 密码安全
- 所有密码均使用 SHA256 哈希存储
- 不会以明文形式保存在数据库中
- 建议测试时使用简单密码（如123456）

### 测试环境注意事项
- 这是测试系统，不要使用真实个人信息
- 测试用户密码统一使用 `123456` 便于记忆
- 如需重置数据库，删除 `campus_secondhand.db` 文件

## 🔄 重置数据库

如果需要重新开始测试，可以重置数据库：

### 方法1：删除数据库文件
```bash
# 停止服务器
# 删除数据库文件
rm campus_secondhand.db  # Linux/Mac
del campus_secondhand.db  # Windows

# 重新启动服务器（会自动创建新数据库）
python start_server.py
```

### 方法2：使用SQL工具
```sql
-- 清空所有表数据（保留表结构）
DELETE FROM orders;
DELETE FROM goods;
DELETE FROM users;

-- 重新创建管理员账户需要重启服务器
```

## 📞 常见问题

### Q1: 忘记管理员密码怎么办？
**A:** 管理员密码固定为 `admin123`，如果修改后忘记，需要：
1. 停止服务器
2. 删除数据库文件
3. 重启服务器会重新创建管理员账户

### Q2: 如何批量创建测试账户？
**A:** 使用管理员账户的"生成测试数据"功能，可以一次性创建多个测试用户。

### Q3: 生成的测试用户密码是什么？
**A:** 所有通过"生成测试数据"创建的用户，密码统一为 `123456`

### Q4: 如何修改默认管理员密码？
**A:** 在 `server/database.py` 文件的 `create_default_admin` 方法中修改：
```python
admin_password = hash_password('你的新密码')
```

### Q5: 测试数据会影响正式使用吗？
**A:** 测试数据和正式数据存储在同一数据库中。建议：
- 测试阶段：尽情使用虚拟数据
- 正式使用前：重置数据库
